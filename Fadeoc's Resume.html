关于

我是一个诚实正直的人。

我的英语水平是CET-6，

我接触互联网有十五年或更久，是知乎和豆瓣的深度用户，使用坚果云和 icloud。我对新事物有强烈的好奇心，很久以前，第一时间体验了 Youtube 和 Chrome，甚至当时 Youtube 尚未被 Google 收购。是 Google 搜索和 Gmail 的深度用户，在桌面端使用 Thunderbird 收发邮件。

我在某些方面比较着重于细节。比如文字与文件管理，我都有着一种近乎执拗的洁癖。

我在知乎参与公共编辑，常常为一个标点符号和文字格式忙到凌晨，却仍能体会乐趣。

前端对页面代码恰恰有着同样强烈的如洁癖一般的要求。而能亲手作出爽滑的页面与交互过程，是一种难以言喻的快乐。

前端

我刚刚学习前端初级支持大约20天。我喜欢学以致用。

这是我学习前端之后制作的第二个页面。我用一下午的时间学会使用 Github，并教授别人怎样使用GitHub。

我遍历了 html 和 css 的主要标签、属性和属性值，对 js 仍局限于其功能，并不了解具体内容。

我对 html 的主要想法是语义化，不仅有利于 Dom，而且在我执拗的审美中语义化也是完美的。于是 class 和 id 选择器变成了重中之重，虽然它们的功能很简洁，但正如在 Windows Explorer 中每个人都会管理文件一样，同样简洁，却充满了无限可能。如果能设计出更容易理解具有层次感的 class 和 id 名字——看一眼名字就知道它处于 Dom 的哪个层级，作用如何——也许会使得对未来前端新手进入一个项目的门槛大大降低。

我对 css 的理解是除了语义结构等 html 本身负责的文本部分，绝大部分格式都应该由 css 来完成。

就我目前的能力而言，对我最难的问题是 html 和 css 结合的框架布局，除了很多我没掌握的细节之外，我觉得 css 本身也有一定的局限性，比如对 html 中 ul 等列表标签的垂直居中功能。我不知道 css3 是否有这个功能，目前还未接触。而 margin、padding、position 与 div、span、ul/ol 等的组合性变幻复杂，同样的功能可以有多种实践方式，又有趣，又充满了挑战性和成就性。

我很喜欢魔兽世界官网扁平化与非扁平化混合的设计方式，以及 Webplatform Wiki 右上角的 edit 按钮的交互过程中圆角会改成直角这种不起眼，用起来也感觉不到，确实用户体验非常好的大道无形的细节设计。

对后端的部分，我尚不清楚具体内容，只知道数据传输的过程是前端——php——服务器+数据库。

我对已有的知识很谦恭，但更喜欢思考本质，思考为什么，当一个知识点不能解释为什么的时候，我就会怀疑它的正确与否，尤其在前端领域，太多的半路杀手侃侃而谈，拷贝别人的代码不求甚解，真正精通本质的大牛们传授的教科书一般的正规知识对我而言更有说服力。

以下内容是我在学习过程中思考、求解过的一部分：

  • strong 和 bold 到底有什么区别？
  • q 和 “” 到底应该用哪一个？
  • 背景图片到底该不该 repeat？
  • ID 和 Class 如何选择？
  • 为何网易lofter下面的链接使用了span，而其他网站大多用 ul？

以下内容是我在学习过程中做过的：

  • 制定学习前端的短期计划
  • 访问IE、Safari、Chrome 和 w3c 阅读解决问题的标准
  • 了解什么是 hack ，为什么要 hack。
  • 学习知乎页面的静态部分，发现不懂处马上深入。
  • 简要阅读了各大浏览器的发展历程。
